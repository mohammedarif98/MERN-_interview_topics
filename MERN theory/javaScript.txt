54) what is promises?

	A Promise is an object that represents the eventual completion or failure of an asynchronous operation. It is used to handle asynchronous code, avoid callback hell, and make
	the code cleaner and more readable.

	--> resolve() is used to mark a Promise as fulfilled and pass the result to the next .then().
	--> reject() is used to mark a Promise as failed and pass the error to .catch().
	--> then() .then() is used to handle the successful result of a resolved Promise.
	--> catch() is used to handle errors from rejected Promises or failures in .then().
	--> finally() executes code after a Promise settles, whether resolved or rejected.

	Promise have 3 states: Pending – initial state, Fulfilled – operation completed successfully, Rejected – operation failed

	Different Promiese Methods:
		Promise.all() → is takes multple proise as array and Waits for all Promises to resolve. If even one Promise rejects, the entire Promise rejects.
		Promise.any() → Returns the first fulfilled promise. If all Promises reject, it throws an AggregateError.
		Promise.race() → Returns with the first promise that fininsh, whether it resolves or rejects.. 
		Promise.allSettled() → Waits for all Promises to complete and returns their results, regardless of success or failure.

	Promise chaining:-->  Promise chaining is the process of linking multiple .then() calls so that the result of one asynchronous operation is passed to the next.

57) what is event loop?

	The event loop is a mechanism that continuously checks whether the call stack is empty. When it is empty, the event loop first executes all tasks from the microtask queue,
	and then takes a task from the macrotask (callback) queue and pushes it onto the call stack.

	The callback queue stores asynchronous tasks like timers and browser events. The microtask queue stores high-priority tasks that run before the callback queue.

57.5) what is callback Queue?

	The Callback Queue (Macrotask Queue) stores callbacks from asynchronous operations like setTimeout, setInterval, and DOM event. When the Call Stack becomes empty, 
	the Event Loop moves these callbacks to the Call Stack for execution.

64) waht is event bubling?

	Event bubbling is a mechanism where an event starts from the target element that triggered it and propagates upward through its parent elements in the DOM tree.
	If a button inside a div is clicked, the click event first fires on the button, then on the div, and so on up to the document object.
	Useful for handling events on multiple elements with a single event listener on a parent element.

65) what is event capturing?

	Event capturing is the opposite of bubbling. The event starts from the top of the DOM tree and propagates down to the target element.
	If a button inside a div is clicked, the click event first fires on the document, then on the div, and finally on the button.
