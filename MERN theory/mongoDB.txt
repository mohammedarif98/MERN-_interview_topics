
1) What is a Database?

	A database is an organized collection of data that allows storing, retrieving, updating, and deleting information efficiently. Databases help applications manage data securely and efficiently.

2) What is DBMS?
	
	DBMS is software used to store, manage, and retrieve data efficiently. It provides features like data security, consistency, and easy querying. Examples include MySQL and MongoDB.
	Types of DBMS : Relational DBMS (RDBMS), NoSQL DBMS

3) What is a Relational Database (RDBMS)?

	RDBMS (Relational Database Management System) is a type of DBMS where data is stored in tables (rows and columns), and relationships between tables are maintained using primary keys
	and foreign keys.

4) What is a NoSQL Database?

	A NoSQL database is a non-relational database designed to store unstructured data. It does not have fixed schemas and supports flexible data models like documents and
	key-value pairs. NoSQL databases are optimized for horizontal scalability, high performance, and handling large volumes of distributed data.

5) What is a Schema?

	A schema is the structure of a database. It defines how data is organized such as tables, fields, data types, relationships, constraints, and indexes.

6) What is SQL?
	
	SQL (Structured Query Language) is a standard language used to communicate with relational databases. It is used to create, read, update, and delete data stored in tables.
	SQL is mainly used with MySQL, PostgreSQL, Oracle

7) What is MySQL?

	MySQL is an open-source Relational Database Management System (RDBMS) that uses SQL to store, retrieve, and manage data in tables with rows and columns.
	It follows ACID properties, supports indexes, joins, constraints, and is widely used in web applications.

8) what is mongoDB? 

	MongoDB is a NoSQL, document-oriented database that stores data in JSON-like documents (BSON) instead of tables and rows.
	It supports a flexible schema, making it ideal for applications where data structure changes frequently.

9) What is a Collection in MongoDB?
	
	A collection is a group of documents in MongoDB. It is similar to a table in a relational database, but it does not enforce a fixed schema.
	Documents within a collection can have different structures and fields.

10) What is a Document in MongoDB?

	A document is a single record of MongoDB collection. It is stored in JSON-like format (BSON) and consists of key–value pairs.
	Each document can have different fields, making it flexible and schema-less.

11) Advantages of MongoDB?

	Flexible Schema, Stores Data as JSON-like Documents, High Performance, Horizontal Scalability

12) Disadvantages of MongoDB?

	No Joins (Limited Relational Capability), Higher Memory Usage, Data Redundancy (Storing the same data multiple times in different places.), Weaker ACID transactional behavior

13) Data Types in MongoDB?

	MongoDB supports BSON data types like String, Number, Boolean, Array, Nested Object, ObjectId, Date, Null, Binary, and Regex.

24) what is ACID Theorem?

	ACID defines rules that ensure reliable and safe database transactions. ACID properties ensure that database transactions are atomic, consistent, isolated, and durable.

	A – Atomicity : A transaction either completes fully or fails completely.If any part of the transaction fails, the entire transaction is rolled back.
	C – Consistency : After every transaction, the database must move from one valid state to another valid state, Making sure data stays correct and follows all rules.
	I – Isolation : Multiple transactions can run at the same time without affecting each other. One transaction cannot see another’s partial data.
	D – Durability : Once a transaction is successfully committed, it is permanently saved. Data remains safe even after a crash or power failure.

33) waht is CAP theorm? 

	CAP Theorem says a distributed system can provide only two of Consistency, Availability, and Partition Tolerance at the same time
	   C – Consistency = All users see the same, up-to-date data at the same time.
	   A – Availability = Every request gets a response, even if some servers are down.
	   P – Partition Tolerance = The system continues to work even if the network fails between servers.

38) what is constraint ?

	A constraint is a rule applied to a database column to ensure data accuracy and integrity. eg: Primary Key, Foreign Key, NOT NULL, UNIQUE, CHECK

	PRIMARY KEY is automatically created when a new row is inserted. It uniquely identifies each row in a table and does not allow duplicate or NULL values
	FOREIGN KEY is used to create a relationship between two tables. It ensures that the value in one table must exist in the referenced table, maintaining referential integrity.
		-----> userId INT, FOREIGN KEY (userId) REFERENCES users(id)
	UNIQUE constraint ensures that all values in a column are different. It allows NULL values, but no duplicate values. -----> email VARCHAR(100) UNIQUE
	NOT NULL constraint ensures that a column cannot contain NULL values. A value must be provided when inserting data. -----> name VARCHAR(50) NOT NULL
	CHECK constraint is used to limit the values that can be stored in a column by applying a condition. ----> age INT CHECK (age >= 18)

39) database relationships ?

	Relationships define how data in one table/collection is connected to another.

 	One-to-One (1:1) : One record in Table A is linked to one record in Table B. Example: User ↔ UserProfile
	One-to-Many (1:N) : One record in Table A can be linked to multiple records in Table B. Example: User → Orders, Author → Blogs
	Many-to-One (N:1) : Many records in Table A are linked to one record in Table B. Example: Many Orders → One User, Many Employees → One Department
	Many-to-Many (M:N) : Many records in Table A are linked to many records in Table B. Example: Students ↔ Courses, Users ↔ Roles
