4) What are Components?

	Components are the building blocks of a React application. They are reusable, independent pieces of UI that accept props as input and return JSX
	Functional components are commonly used with hooks, while class components are older and uses on lifecycle methods.

5) what is Fucntional Components?

	A functional component is the modern way to define components in a React application. It is a JavaScript function that accepts props and returns JSX. 
	Functional components are simpler, easier to read, and use React Hooks to manage state and lifecycle features (Side effects by useEffect hook).

	In functional components, the useEffect hook is used to handle lifecycle features. Empty dependency array [] acts as componentDidMount, a value in the array acts as componentDidUpdate,
	and the return function acts as componentWillUnmount

	React component lifecycle has three phase:
		Mounting (Component Created): component is created and added to the DOM.
		Updating (State or Props Change): when compoenent is re-render due to state or props change.
		Unmounting (Component Removed): when the component is removed from the DOM, Managed using cleanup function

6) waht is Class Component?

	A class component is older way of defining components in React. It is created using an ES6 class that extends React.Component, Class components can manage state and handle lifecycle
	methods. They use lifecycle methods such as componentDidMount, componentDidUpdate, and componentWillUnmount to control behavior during different phases of a component’s lifecycle.

7) what is LifeCyle method?

	Lifecycle methods control a component’s behavior during mounting, updating, and unmounting, handling side effects such as data fetching and cleanup.

	ComponentDidMount: its only run one time after the first render.
	ComponentDidUpdate: its runs after state/props change.
	ComponentWillUnmount: Runs just before the component is removed from the DOM.(cleanup).
	
	React component lifecycle has three phases — mounting, updating, and unmounting. In class components these are handled by componentDidMount, componentDidUpdate,
	and componentWillUnmount. In functional components, all three are handled by the useEffect hook.

	Props drilling is the process of passing data from a parent component to deeply nested child components through intermediate components that don’t need the data,

22) what is lazy loading?

	Lazy loading is a technique where components or resources are loaded only when they are needed, instead of loading everything at once.
	its Improves initial load time, Reduces bundle size, Enhances performance, Implemented using React.lazy() and Suspense

	Component-Level Lazy Loading: Component-level lazy loading loads a component only when it is rendered, using React.lazy and Suspense.

	Route-Level Code Splitting: Route-level code splitting splits the application bundle by routes, so only the required code for the current route is downloaded, improving performance.

25) what is redux?

	Redux is a predictable state management library for JavaScript applications used to manage application state in a centralized store.

	Core Concepts (principle) of Redux: 
		--> Store → Holds the entire application state in a single centralized object.
		--> Action → Plain js object that describes what happened
		--> Reducer → Reducer is a pure function that takes the current state and an action, and returns a new state without mutating the original state
		--> Dispatch → Dispatch is a function that triggers a state change, it sends an action to the store, which forwards it to the reducer.

	How Redux Works: In Redux component dispatches an action describing what happened. The action is sent to the Redux store, which forwards it to the reducer.
	The reducer returns a new state based on the action type. The store updates its state, and connected components receive the updated state and re-render if needed.

	Middleware: middleware in Redux is a function that sits between dispatching an action and the reducer, allowing you to intercept, modify, delay, or handle side effects before
		    the action reaches the reducer.

	Common Use Cases: Handling async actions (API calls), Logging actions, Error handling, Analytics, Authentication logic.
	Common Redux Middleware: redux-thunk, redux-saga, redux-logger 

26) how react application works?

	When a React application starts, the browser loads index.html, which contains a root element (e.g., <div id="root"></div>). Then the entry file (main.jsx) executes and renders
	the root component (App) into the DOM using ReactDOM.createRoot().render(). React builds a Virtual DOM from the component tree. When state or props change, React creates a new
	Virtual DOM, compares it with the previous one using a diffing algorithm, and updates only the changed parts in the real DOM through reconciliation, ensuring efficient UI updates.

51} why direct mutating objects is dangerous?

	Directly updating state is dangerous in React because it keeps the same reference. React works by detecting reference changes to detect updates, it may skip re-rendering, leading to
	stale UI and hard-to-debug bugs.

	-----> const [user, setUser] = useState({ name: "Arif", age: 25 });     // ❌ Directly mutating state
	       user.name = "Rahul";
	       setUser(user);

	-----> setUser(prev => ({...prev, name: "Rahul"}));                  // Correct Way (Immutable Update)
