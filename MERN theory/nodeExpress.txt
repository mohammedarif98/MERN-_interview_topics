

1) What is Node.js?

	Node.js is a runtime environment that allows JavaScript to run outside the browser, mainly on the server. It is built on Google Chrome’s V8 engine and is non-blocking and event-driven,
	which makes it fast and scalable. a runtime environment is where code is executed, and it provides memory, libraries, and system APIs needed during execution.

	Express.js is a lightweight web framework built on top of Node.js. It simplifies building web servers and APIs by providing features like routing, middleware, and request/response handling.

2) Why is Node.js non-blocking?

	Blocking: Blocking in Node.js means an operation stops the execution of other code until it completes, blocking the event loop.

	Non-Blocking: Non-blocking means Node.js does not wait for an operation to finish before moving on to the next task. When tasks like database queries or file reads happen,
			Node continues executing other code instead of stopping. This prevents the application from freezing or slowing down.

37) what is JWT?

	JWT (JSON Web Token) is a compact, secure token format used for authentication and authorization in web applications. It allows the server to verify a user’s identity
	without storing session data.

	JWTs are commonly used for: Login authentication, API authorization, Protecting routes

	How its Work:  When a user logs in, they send their user credentials to the server. The server validates these credentials, and if they are correct, it creates a JWT, token
		       that contains user information, a secure signature, and an expiry time. The server then sends this token back to the client.The client stores the token
		       (either in localStorage or an HTTP-only cookie). Whenever the client accesses a protected API, it includes this token in the request, usually in the Authorization header.
		       The server verifies the token to ensure it hasn’t been tampered with and hasn’t expired. This verification happens without querying the database. If the token is valid,
                       the server grants access to the requested resource.
	
	JWT Structure (Three Parts) : header.payload.signature. header Contains token type + hashing algorithm, payload contain user data and signature Used to verify token authenticity

82) what is difference between a Load Balancer and an API Gateway?

83) What is a Webhook?

	A webhook is a mechanism that allows one system to automatically send real-time data to another system via an HTTP request when a specific event occurs.
	Instead of repeatedly requesting data, the system automatically sends an HTTP request (usually POST) to a predefined URL with event details.
	
	The frontend initiates the payment, the backend creates an order, the payment gateway processes it, and the final payment status is confirmed through webhooks

	The payment process starts when a user clicks Pay on the frontend, which sends a request to the backend. The backend creates a payment order and sends it to the payment gateway.
	The user completes the payment, and the gateway processes it. Finally, the gateway sends a webhook to the backend to confirm the payment, and the backend updates the payment status.
	The actual use of a webhook in the payment process:  A webhook ensures the backend always knows the real payment result, even if the frontend never receives a response.
